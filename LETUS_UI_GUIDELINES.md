# LETUS UI GUIDELINES

이 문서는 LETUS 고객사 관리 플랫폼의 공통 UI 컴포넌트 디자인 규칙을 정리한 문서입니다. 새로운 메뉴나 컴포넌트를 생성할 때 **반드시** 이 규칙을 준수하여 디자인의 통일성을 유지해야 합니다.

## 2. 컴포넌트별 디자인 규칙 (Component Rules)

### 2.1 주요 액션 버튼 (Action Buttons)
- 시스템에 데이터를 추가, 수정, 저장하는 기능이 포함된 모든 버튼은 **LETUS BLACK** 배경에 **흰색 텍스트** (예: `bg-[#414344] text-white hover:bg-[#414344]/90`) 형식을 갖춰야 합니다.
- 삭제 등 위험한 동작이더라도, 일관성을 위해 기본 테마는 BLACK으로 하되 내부 아이콘에만 붉은 색을 주어 구분할 수 있습니다.
- (적용 예: 신규 고객사 등록, 리포트 다운로드, 코스트센터 추가 버튼 등)

### 2.2 메뉴 탑 헤더 (Top Header)
- 상단 패널은 **LETUS ORANGE (`bg-[#ff5c39]`)** 배경으로 적용하며, 흰색 텍스트로 가독성을 높입니다.
- 선택된 메뉴의 대메뉴명은 크고 명확하게 왼쪽 상단에 표기해야 합니다.

### 2.3 표 (Table Data)
- **표의 제목행(Table Header Row)**: 모든 데이터 테이블 컴포넌트의 헤더 로우(`thead > tr` 요소 혹은 `TableHeader` 컴포넌트)의 배경색은 **LIGHT GREEN (`bg-[#e1ebe0]`)** 로 통일해야 합니다.
- **표 제목행 가운데 정렬**: 표의 제목을 나타내는 모든 셀(`<th>`, `TableHead`)은 글자가 **가운데 정렬(`text-center`)** 되도록 일관성을 유지합니다.
- **표의 테두리(Borders)**: 행과 행 사이뿐만 아니라 **열과 열 사이(세로 테두리)**에도 동일한 구조의 옅은 회색 선(`border-slate-200`)을 적용해 스프레드시트처럼 명확하고 깔끔하게 데이터를 구분합니다.
- **폰트 크기(Font Size)**: 표의 제목열 폰트는 모든 메뉴에 걸쳐 **`text-sm` (14px)**로 통일하고, 표 내의 데이터 내용 폰트 크기는 제목열보다 한 단계 작은 **`text-xs` (12px)**로 적용하여 위계와 정보 가독성을 지킵니다.
- (예: Shadcn UI의 `Table` 단위에서 `text-xs` 베이스를 잡고, `TableHead` 에는 `text-sm`를 적용하는 방식으로 구현)
- 테이블 컨테이너 영역은 흰색 바탕(`bg-white`)으로 한정지어 시각적으로 깔끔한 느낌을 줍니다.

### 2.4 데이터 필터 박스 (Data Filters)
- 페이지 상단에 위치하는 필터(정산월, 고객사 선택 등) 컨트롤러는 상하 스택 뷰 대신 **가로형 배치(Inline, Flex row)**를 기본으로 합니다. 
- 직관적인 사용을 위해 드롭다운 등의 경우엔 "정산월 [ ▽ ]" 과 같이 간결하게 노출합니다.

> **작업 시 주의사항 (AI 가이드라인)**:  
> 새로운 화면을 디자인하거나 기존 컴포넌트를 수정할 때, 임의의 회색/파란색 테마(`bg-blue-50`, `bg-slate-50` 등)가 아닌 이 문서에 기재된 LETUS 브랜드 색상을 최우선적으로 적용하세요.
